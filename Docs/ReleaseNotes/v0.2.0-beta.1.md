# Release Notes - v0.2.0-beta.1

## Overview
This beta release introduces advanced parameter tracking and interpolation capabilities, enhancing the precision and natural feel of face tracking transformations.

## üöÄ What's New

### Runtime Extremums Tracking
- **Real-time Min/Max Monitoring**: Tracks minimum and maximum values observed for each PC tracking parameter during runtime
- **Parameter Table Integration**: New "Min/Max" column displays current extremum ranges in the PC parameter table
- **Persistent Tracking**: Extremums are maintained across transformation cycles and reset when rules are reloaded
- **Visual Feedback**: Extremum ranges help users understand the actual value ranges their parameters are experiencing
- **Configuration Support**: Min/Max column can be enabled/disabled via UserPreferences.json parameter table configuration

### Custom Interpolation Curves
- **Bezier Curve Support**: Introduces Bezier interpolation for natural parameter responses
- **CSS-Compatible Format**: Control points use compact array format matching CSS `cubic-bezier()` syntax
- **Online Tool Integration**: Compatible with [cubic-bezier.com](https://cubic-bezier.com/) and other CSS animation generators
- **Multiple Curve Types**: Support for ease-in, ease-out, ease-in-out, and custom curve patterns
- **Performance Optimized**: Supports 2-8 control points with De Casteljau's algorithm implementation
- **Configuration Integration**: Interpolation methods defined in transformation rules with type and control points

### Enhanced Parameter Table Display
- **New Columns**: Added "Min/Max" and "Interpolation" columns to PC parameter table
- **Dynamic Configuration**: Users can customize which columns to display via UserPreferences.json
- **Interpolation Information**: Shows interpolation method (e.g., "Linear", "Bezier (0, 0.5, 0.5, 1)") for each parameter
- **Extremum Ranges**: Displays current min/max values observed during runtime tracking

## üîß Technical Changes

### New Models
- **ParameterExtremums**: Tracks minimum and maximum values for parameters with initialization state
- **BezierInterpolation**: Represents Bezier curve interpolation with variable control points (2-8 points)
- **Point**: Represents 2D points for Bezier curve control points

### New Services
- **BezierInterpolationMethod**: Implements Bezier curve evaluation using De Casteljau's algorithm
- **InterpolationMethodFactory**: Factory for creating interpolation methods from configuration

### Enhanced Interfaces
- **IInterpolationMethod**: Interface for pluggable interpolation methods
- **IInterpolationDefinition**: Base interface for interpolation definitions
- **ParameterTableColumn**: Added MinMax and Interpolation column options

### Updated Components
- **TransformationEngine**: Integrated extremum tracking and interpolation support
- **PCTrackingInfo**: Enhanced with extremum and interpolation data
- **PCTrackingInfoFormatter**: Added formatting for new table columns
- **ParameterTableConfigurationManager**: Updated default columns to include new options

## üìã Configuration Examples

### Bezier Interpolation Configuration
```json
{
  "name": "EyeOpenLeft",
  "func": "EyeOpenLeft",
  "min": 0.0,
  "max": 1.0,
  "defaultValue": 0.5,
  "interpolation": {
    "type": "BezierInterpolation",
    "controlPoints": [0.42, 0, 1, 1]
  }
}
```

### Common Bezier Curve Patterns
**Ease-in (accelerating):**
```json
"controlPoints": [0.42, 0, 1, 1]
```

**Ease-out (decelerating):**
```json
"controlPoints": [0, 0, 0.58, 1]
```

**Ease-in-out (smooth):**
```json
"controlPoints": [0.42, 0, 0.58, 1]
```

### Parameter Table Customization
Configure which columns to display in UserPreferences.json:
```json
{
  "PCParameterTableColumns": ["ParameterName", "ProgressBar", "Value", "MinMax", "Interpolation"]
}
```

## üêõ Troubleshooting

If you encounter issues:
1. **Extremums not updating**: Ensure transformation rules are properly configured
2. **Bezier curves not working**: Verify control points array format matches CSS cubic-bezier syntax
3. **Parameter table display issues**: Check UserPreferences.json column configuration
4. **Performance issues**: Consider reducing number of control points for complex Bezier curves
