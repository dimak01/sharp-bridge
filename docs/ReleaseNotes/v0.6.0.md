# Sharp Bridge v0.6.0

## What's New

- **Default parameter protection**: Parameter sync now retrieves the list of default VTube Studio parameters from your loaded model and compares each transformation rule name to that list. Rules whose names match a default parameter (e.g. `FaceAngleX`, `EyeOpenLeft`) are left unchanged—Sharp Bridge does not create or update them as custom parameters. Only names that do not match defaults are created or updated as custom parameters. You can use default parameter names directly in your transformation configuration.
- **Transformation rule name validation**: Transformation rule names must be between 4 and 32 characters (VTube Studio API requirement). Configurations with names outside this range are rejected during validation.

## Bug Fixes

- **DeleteParameterAsync** now validates the `parameter` argument and throws `ArgumentNullException` when it is null, preventing `NullReferenceException` in error paths.

## Breaking Changes

- **Configurable parameter prefix removed**: The parameter prefix option has been removed from VTube Studio PC configuration. Use default VTS parameter names directly in transformation rules (e.g. `"name": "FaceAngleX"`), or use unique names for custom parameters (e.g. `"name": "CustomFaceAngleX"`). See the User Guide → Transformations → *Parameter Names and Default Parameter Protection* for details.

## Migration Guide

If you previously used a **parameter prefix** in your VTube Studio PC config:

1. Remove the `parameterPrefix` (or equivalent) field from your configuration.
2. In your transformation rules, use the actual VTS parameter names in the `"name"` field:
   - For parameters that exist as defaults on your model (e.g. `FaceAngleX`, `EyeOpenLeft`), use that exact name—Sharp Bridge will use the existing default parameter.
   - For parameters that should be custom, use any unique name that does not match a default (e.g. `MyCustomParam`). Sharp Bridge will create or update them as custom parameters.

See [Transformations — Parameter Names and Default Parameter Protection](https://github.com/dimak01/sharp-bridge/blob/main/docs/UserGuide/AdvancedFeatures/Transformations.md#parameter-names-and-default-parameter-protection) in the User Guide for full details.
